    Sip

    The SIP class takes care of all SIP communication in the background. Currently this is used to check the presence of contacts.

    Constructor

    new Sip(app)

    Parameters:
    Name Type Description
    app ClickToDialApp

    The application object.

    Methods

    connect()

    Connect SipML5 to the websocket SIP backend.

    disconnect(reconnect)

    Graceful stop, do not reconnect automatically.

    Parameters:
    Name Type Default Description
    reconnect Boolean true

    Whether try to reconnect.

    initStack()

    Init and start a new stack.

    sipStatusEvent(e)

    The SIP stack fires a new event, which is handled by this function.

    Parameters:
    Name Type Description
    e Event

    Catch-all event when SipML UA tries to connect to the websocket proxy.

    subscribeEvent(e, accountId, resolve)

    Called multiple times for each presence subscription is registered.

    Parameters:
    Name Type Description
    e Event

    Catch-all event when a subscribe event is triggered.
    accountId Number

    The accountId of the subscriber.
    resolve function

    The Promise resolver function.

    subscribePresence(accountId) → {Promise}

    Does the actual subscription to the SIP server.

    Parameters:
    Name Type Description
    accountId Number

    Account Id of VoIP-account to subscribe to.
    Returns:
    Promise -
    • Resolved when the subscription is ready.

    unsubscribePresence(accountId)

    Stop listening for subscriber events from the SIP server and remove the cached subscriber state.

    Parameters:
    Name Type Description
    accountId Number

    The accountId to deregister.

    async updatePresence(refresh)

    Update presence information for given account ids. The presence info is cached and will only subscribe for account ids that are not yet in the cached states object. This behaviour can be overridden using the refresh option. With refresh, all supplied account ids will have their presence updated from the SIP server.

    Parameters:
    Name Type Description
    refresh Boolean

    Force refreshing presence from the sip service.
